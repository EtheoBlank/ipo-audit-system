# IPO审计系统 - 数据接口规范

本文档说明系统支持的数据格式，以及如何从各主流ERP系统导出数据。

---

## 一、支持的数据类型

### 1.1 科目余额表 (Account Balance)

**用途**: 记录各会计科目的期初余额、本期发生额、期末余额

**标准格式字段**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 科目编码 | string | ✅ | 会计科目编码，如1001 |
| 科目名称 | string | ✅ | 会计科目名称，如"银行存款" |
| 余额方向 | string | ✅ | 借/贷 |
| 期初余额 | number | | 期初余额(默认为0) |
| 借方发生额 | number | | 本期借方发生额合计 |
| 贷方发生额 | number | | 本期贷方发生额合计 |
| 期末余额 | number | | 期末余额 |

---

### 1.2 序时账 (Chronological Account)

**用途**: 记录每笔凭证的分录明细

**标准格式字段**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 凭证日期 | string/date | ✅ | 凭证日期，格式YYYY-MM-DD |
| 凭证号 | string | ✅ | 凭证编号 |
| 科目编码 | string | ✅ | 会计科目编码 |
| 科目名称 | string | ✅ | 会计科目名称 |
| 借方金额 | number | |借方金额(默认为0) |
| 贷方金额 | number | | 贷方金额(默认为0) |
| 摘要 | string | | 凭证摘要 |
| 辅助核算 | string | | 辅助核算项目 |

---

### 1.3 银行对账单 (Bank Statement)

**用途**: 记录银行账户的交易明细和余额

**标准格式字段**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 对账日期 | string/date | ✅ | 对账日期 |
| 凭证号 | string | | 关联凭证号 |
| 描述 | string | ✅ | 交易描述 |
| 借方金额 | number | | 借方发生额(支出) |
| 贷方金额 | number | | 贷方发生额(收入) |
| 余额 | number | ✅ | 账户余额 |
| 银行账号 | string | | 银行账号 |

### 2.5 SAP S/4HANA

**导出路径**: 财务会计 → 总账 → 信息系统 → 报表

**常用事务代码**:
- **FS10N**: 科目余额查询
- **FAGLB03**: 总账报表(含银行对账单)
- **FB03**: 显示凭证
- **FAGLE**: 凭证列表

**字段对照表**:

| SAP字段 | 标准字段 | 备注 |
|---------|----------|------|
| SAKNR | 科目编码 | |
| KTOKS / TOPL | 科目描述 | |
| DRCRK | 借贷标识 | S=借方(Soll)，H=贷方(Haben) |
| TSL / TSL_1 | 期末/期初余额 | 本位币金额 |
| HSL | 本年累计借方 | 本位币 |
| KSL | 本年累计贷方 | 本位币 |

**序时账导出** (通过ACDOCA表):

| SAP字段 | 标准字段 | 备注 |
|---------|----------|------|
| BLDAT | 凭证日期 | |
| BUDAT | 过账日期 | |
| BELNR | 凭证编号 | |
| BKTXT | 凭证抬头文本 | |
| RACCT | 科目编号 | |
| WSL | 金额 | 借方为正，贷方为负 |
| PRCTR | 利润中心 | |
| AWTYP | 参考交易类型 | |

**银行对账单导出** (FAGLB03):

| SAP字段 | 标准字段 | 备注 |
|---------|----------|------|
| BUKRS | 公司代码 | |
| BANKN | 银行账号 | |
| HKDAT | 对账日期 | |
| WRSBE | 收入金额 | |
| WRSBR | 支出金额 | |

**SAP导出注意事项**:
1. SAP金额可能包含多位小数，需确保精度
2. 外币金额需使用本币金额(TSL)进行审计
3. 凭证日期(BLDAT)与过账日期(BUDAT)可能不同
4. SAP支持跨公司代码合并报表

## 四、合并报表数据格式

系统支持**合并报表**与**单体报表**的对比分析。

**合并报表格式**: 与科目余额表格式相同，但数据为合并抵消后的汇总数据。

**附加字段**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| is_consolidation | boolean | 是否为合并报表数据 |
| internal_elimination | number | 内部交易抵销金额 |

**合并报表与单体报表差异说明**:
- 内部交易抵销(资产) = 单体资产 - 合并资产
- 内部交易抵销(负债) = 单体负债 - 合并负债
- 差异主要来源于母子公司之间的关联交易

## 六、常见问题

###6.1 金额精度问题
**问题**: 导入后金额与原数据有微小差异
**解决**: 系统使用浮点数，建议ERP导出时保留2位小数

### 6.2 日期格式问题
**问题**: 日期无法正确解析
**解决**: 统一使用YYYY-MM-DD格式，或在Excel中设置单元格格式为日期

### 6.3 辅助核算缺失
**问题**: 序时账中缺少辅助核算信息
**解决**: 确保ERP导出时包含辅助核算字段，或在导出设置中勾选相关选项

### 6.4 SAP外币金额
**问题**: SAP导出外币金额与本币金额混淆
**解决**:审计时使用本币金额(TSL)，确保科目币种一致

### 6.5 金蝶多币种
**问题**: 金蝶多账套/多币种数据如何处理
**解决**: 按账套和币种分别导出，系统支持多账套数据汇总
